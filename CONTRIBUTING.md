# Contributing

Thanks for your interest in contributing.

## Quick start

1. **Open an issue first** for non-trivial changes — this avoids wasted work if the idea needs discussion.
2. **Fork** and create a branch using the prefixes in [CLAUDE.md](CLAUDE.md): `feat/`, `fix/`, `refactor/`, `docs/`, `chore/`, `test/`, `style/`, `perf/`.
3. **Open a pull request** using the template. Keep PRs small and focused on a single concern.

## Code style and conventions

Project conventions — branch naming, commit messages, PR format, code structure, testing, scope discipline — live in [CLAUDE.md](CLAUDE.md). They apply equally to human contributors and AI-assisted work; treat that file as the source of truth.

## Reporting bugs and requesting features

Use the [issue templates](.github/ISSUE_TEMPLATE/). For security vulnerabilities, see [SECURITY.md](SECURITY.md) — do not file a public issue.

## Code of Conduct

By participating you agree to uphold the [Code of Conduct](CODE_OF_CONDUCT.md).
